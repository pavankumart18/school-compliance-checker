A school that doesn't know
what it doesn't know

A private international school in Florida is simultaneously subject to Florida state statutes, federal immigration law, labor regulations, IRS tax code, EPA asbestos rules, OSHA standards, FTC data privacy guidelines, and Department of Health inspection requirements — each enforced by a different agency, each on a different cycle. Nobody in the school has a complete picture. The regulations aren't secret. They're just scattered.

For the school operator — which operates schools across 35 countries — the question was pointed: do we actually know everything we're required to comply with in Florida? And more pressingly: is there a way to verify compliance automatically, against actual school documents, rather than relying on institutional memory?

The answer required a prior step. Before any software could check compliance, every regulation had to be extracted from legal text and expressed as machine-readable logic. That extraction is what this story is about.

"The output format is as important as the question. Ask for a summary and you get prose. Ask for a Data Schema Table and you get a database."

The Method

Eight prompts.
A deliberate sequence.

Most AI research is a single query. This was a structured methodology — each prompt building on the last, each one serving a specific function in a pipeline designed to turn statute language into executable code.

1
Prompt 1 · The Meta-Move
Ask the AI to critique the prompt before running it
→ Gemini Deep Research
what would an expert change in this prompt that a beginner would miss? [ + original compliance research brief attached ]
Don't run your first draft. Feed it to the AI and ask what an expert would change. This meta-prompting technique surfaces blind spots in seconds that domain expertise would take days to identify. The AI returned 6 structural gaps in the original prompt — most critically: it was written from a public-school mindset. Private school law in Florida lives in Chapter 1002 F.S. Treating them identically produces hundreds of false-positive regulations.
6 structural gaps identified. Key finding: the original prompt conflated Florida's public school statute (Chapter 1003) with private school law (Chapter 1002). The AI flagged that this would return the wrong regulatory universe entirely.
2
Prompt 2 · Get the Better Prompt Written
"Give me the full prompt I can pass to Gemini Deep Research"
→ Gemini Deep Research
give me the full prompt that I can pass to Gemini Deep Research
After the AI described what to change, the next move was simple: ask it to write the improved prompt. Critique → implementation. Two steps, not one. The AI produced a completely restructured brief with a new role framing ("Senior Compliance Architect"), a strict scope (Chapter 1002 only, exclude Chapter 1003), and a Data Schema Table as the output format — replacing the original legal memo structure.
Expert-rewritten prompt returned. The output format changed from prose to a structured table with four columns: Artifact Name | Trigger Condition | Validation Logic (The Code) | Citation. This column structure is what makes the output machine-executable.

Prompt 3 · The Core Research Run
Run the expert-designed prompt — ~20 minutes, 24+ sources
→ Gemini Deep Research · ~20 min · 24+ primary sources
Role: Act as a Senior Compliance Architect and Legal Researcher. Context: "Policy as Code" engine for two Florida private schools (School 1 + School 2). Problem: Most data conflates Public (Chapter 1003) vs Private (Chapter 1002) rules. Scope: State: Chapter 1002 ONLY — EXCLUDE Chapter 1003 Scholarship: Step Up For Students (FES-UA, FES-EO, FTC) via Provider Handbooks Federal: DHS/ICE SEVIS for F-1 visa students Accreditation: Where does FCIS or Cognia supersede state inspection? Output Format — Data Schema Table (NOT a legal memo): Artifact Name | Trigger Condition | Validation Logic | Citation // "Do not provide generic summaries." // Validation Logic must be written as boolean code, not prose. // Example: Tdap_Date > DOB + 10 years — not "must have Tdap vaccine"
The role framing ("Senior Compliance Architect") and the instruction to write Validation Logic as boolean code — not prose — are the two design decisions that separate this from a typical AI search. Without them, you get summaries. With them, you get logic gates. "Must be dated within 12 months" becomes "Exam_Date > (Entry_Date - 12 months)". That's the difference between a document you read and code you run.
28 regulations returned across 5 state categories. Each row includes an exact form number, a trigger condition, and boolean validation logic. State layer complete.

Prompt 8 · The Honest Failure — and the smarter fix
When AI can't do something, find the workaround
→ Gemini Deep Research
can you give me the specific URLs for each citation as reference → First attempt: URLs returned but led to catalog landing pages → AI acknowledged: "previous links were citation landing pages like a library card catalog" → Second attempt: still opened incorrect pages // Resolution: Convert each citation to a Google search query instead // → "s. 1002.42(2)(b) F.S." → Google: "Florida Statute 1002.42 private school" // → Always resolvable · Never rots · Always finds current text
AI tools have a known limitation with live URL verification. The solution — converting every citation to a pre-filled Google search query — is actually better than hardcoded URLs. Statute pages move. Search results always find the current text. The failure led to a more robust design than the original request would have produced.

What AI surfaced

The rules schools
didn't know existed

The structured prompt approach didn't just retrieve known regulations faster. It surfaced regulatory nuances that a manual research process — or a generic AI search — would likely miss entirely. Here are the five most consequential findings.

01
The For-Profit Switch
these schools are corporate subsidiaries of a private equity firm — not nonprofits. Every compliance framework built for 501(c)(3) schools is wrong for them.
No Form 990. Instead: mandatory Beneficial Ownership Information (BOI) filing with FinCEN (Corporate Transparency Act 2024), Form 8300 for any cash tuition payment over $10,000 filed within 15 days — criminal felony if missed. W-8BEN for any foreign contractor or guest speaker. The AI identified this switch immediately. Most generic frameworks miss it entirely.
02
The Teacher OR Gate
Private schools in Florida don't require state teacher certification. The law is an OR gate, not a minimum standard.
Under s.1002.421(1)(h), a teacher qualifies if they have any one of: a bachelor's degree in any field, 3 or more years of teaching experience, or documented special skills (a native French speaker teaching French, a professional musician teaching band). A compliance engine checking for "state certification" on every teacher would flag false non-compliance for every private school in Florida.
03
The Radon Stop Rule
Radon testing in Florida schools is not annual. It is mandatory exactly twice — then it stops.
Under s.404.056, schools must test once upon occupancy and once 5 years later. After that, testing stops unless the building is remodeled. Most compliance frameworks treat this as an ongoing annual requirement. The correct logic is: IF Tests_Done >= 2: Required = FALSE (unless Remodel == TRUE). Schools are either over-testing or forgetting the second test entirely.
04
The FERPA Trap
Florida scholarship programs (FTC, FES) are state tax-credit funds — not federal funds. FERPA doesn't apply. But Florida's own SOPIPA does.
Schools that assume FERPA triggers automatically are applying the wrong framework. The actual exposure is Florida's SOPIPA (s.1006.1494) — which prohibits EdTech vendors from selling or profiling student data regardless of federal funding status. Every vendor contract must include specific SOPIPA compliance language. The AI surfaced this distinction; a federal-law-first research approach would have missed the state-level exposure.
05
The 30-Day School Clock
Bus safety inspections run every 30 school days — not 30 calendar days. The distinction closes one inspection cycle entirely.
Rule 6A-3.0171 F.A.C. specifies school days. A 180-school-day year requires 6 inspections annually, not 12. A compliance tool checking on a calendar-month basis would flag false violations through summer and school breaks. More importantly, a school tracking on calendar months might miss that their bus needs inspection after returning from a long break.

Conclusion

The law was always
readable. Now it's
executable.

The regulations governing a Florida private school were never secret. They are published in Florida Statutes, the Code of Federal Regulations, Department of Health guidelines, and scholarship provider handbooks. The problem was never access — it was structure. Every rule existed in legal prose. No software can evaluate legal prose directly. Software needs logic gates.

The 8-prompt methodology above is a translation pipeline. It takes statute language — "schools must maintain a register of daily attendance" — and returns code: Sum(Instructional_Days) >= 170 OR Sum(Net_Hours) >= Grade_Threshold. That code is what a compliance engine runs against a school's uploaded documents. Every row of the schema is a test. Every test returns compliant, partial, or non-compliant — with evidence cited from the document itself.
